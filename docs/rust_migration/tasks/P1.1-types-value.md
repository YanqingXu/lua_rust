## Task Description

Port the C++ foundational type system (`types.hpp`) and `Value` class (`value.hpp` + `value.cpp`) from C++23 to idiomatic Rust, maintaining exact behavioral equivalence with the C++ baseline.

This is the **very first Phase 1 task** — all subsequent migration work depends on these types.

---

## C++ Source Analysis

### 1. `src/common/types.hpp` → `lua_core::types`

**What to port:**

| C++ | Rust | Notes |
|---|---|---|
| `Lua::i8/i16/i32/i64` | `i8`, `i16`, `i32`, `i64` | Rust std already has these |
| `Lua::u8/u16/u32/u64` | `u8`, `u16`, `u32`, `u64` | Rust std already has these |
| `Lua::f32/f64` | `f32`, `f64` | Rust std already has these |
| `Lua::usize/isize` | `usize`, `isize` | Rust std already has these |
| `Lua::LuaNumber` (f64) | `pub type LuaNumber = f64` | Type alias |
| `Lua::LuaInteger` (i64) | `pub type LuaInteger = i64` | Type alias |
| `Lua::ValueType` (enum class : u8) | `#[repr(u8)] pub enum ValueType` | Discriminants MUST match exactly |
| `Lua::GCObjectType` (enum class : u8) | `#[repr(u8)] pub enum GcObjectType` | Discriminants MUST match exactly |
| `Lua::GCColor` (enum class : u8) | `#[repr(u8)] pub enum GcColor` | For GC phase (Phase 1.3) |

**Critical constraints:**
- `ValueType` discriminant values MUST be: Nil=0, Boolean=1, LightUserdata=2, Number=3, String=4, Table=5, Function=6, Userdata=7, Thread=8
- `GcObjectType` discriminant values MUST be: String=4, Table=5, Function=6, Userdata=7, Thread=8, Proto=9, Upval=10
- These values are consumed by other code via `as u8` casts — do not renumber them.

### 2. `src/core/value.hpp` → `lua_core::value`

**What to port:**

```rust
/// Lua 5.1 value — direct mapping of C++ ValueVariant (std::variant)
/// Size target: ≤ 16 bytes on 64-bit (same as C++)
#[derive(Clone, Debug)]
pub enum Value {
    Nil,                              // index 0 — matches ValueType::Nil
    Boolean(bool),                    // index 1 — matches ValueType::Boolean
    LightUserdata(*mut c_void),       // index 2 — matches ValueType::LightUserdata
    Number(f64),                      // index 3 — matches ValueType::Number
    String(GcRef<GcString>),          // index 4 — matches ValueType::String
    Table(GcRef<Table>),              // index 5 — matches ValueType::Table
    Function(GcRef<Function>),        // index 6 — matches ValueType::Function
    Userdata(GcRef<Userdata>),        // index 7 — matches ValueType::Userdata
    Thread(GcRef<Thread>),            // index 8 — matches ValueType::Thread
}
```

**IMPORTANT — Phase 1.1 bootstrap strategy:**
Since `GcRef<T>`, `GcString`, `Table`, `Function`, `Userdata`, and `Thread` don't exist yet in Phase 1.1, use a **type alias placeholder** that will be replaced in Phase 1.2–1.4:

```rust
// In types.rs (temporary — will be replaced in P1.2)
/// Placeholder for GC-managed reference — replaced with real GcRef<T> in Phase 1.2
pub type GcRef<T> = *const T;

// In the relevant modules (temporary)
pub struct GcString;   // Placeholder — implemented in Phase 1.2
pub struct Table;       // Placeholder — implemented in Phase 1.4
pub struct Function;    // Placeholder — implemented in Phase 1.4
pub struct Userdata;    // Placeholder — implemented in Phase 1.4
pub struct Thread;      // Placeholder — implemented in Phase 1.4
```

This allows `Value` to compile now while GC types are built incrementally. The `GcRef<T>` placeholder will be replaced with the real safe wrapper in Phase 1.2.

**`Display` implementation must match C++ `toString()` output exactly:**
- `Nil` → `"nil"`
- `Boolean(true)` → `"true"`, `Boolean(false)` → `"false"`
- `Number(n)` → format with 6 decimal places (e.g., `"3.140000"`)
- `LightUserdata(p)` → `"lightuserdata: 0x..."` (hex pointer)
- `String(_)` → `"string: 0x..."` (hex pointer)
- `Table(_)` → `"table: 0x..."` (hex pointer)
- `Function(_)` → `"function: 0x..."` (hex pointer)
- `Userdata(_)` → `"userdata: 0x..."` (hex pointer)
- `Thread(_)` → `"thread: 0x..."` (hex pointer)

### 3. `Value` enum discriminant guarantee

The C++ code has 9 `static_assert` statements verifying that each `ValueType` variant maps to the correct `ValueVariant` alternative by index. The Rust equivalent is the enum variant order — the **compiler naturally assigns discriminants 0, 1, 2, ...** in declaration order. No need for `#[repr(u8)]` on `Value` itself, but the variant order must match:

```
Value::Nil             → discriminant 0 = ValueType::Nil (0)
Value::Boolean(_)      → discriminant 1 = ValueType::Boolean (1)
Value::LightUserdata(_) → discriminant 2 = ValueType::LightUserdata (2)
Value::Number(_)       → discriminant 3 = ValueType::Number (3)
Value::String(_)       → discriminant 4 = ValueType::String (4)
Value::Table(_)        → discriminant 5 = ValueType::Table (5)
Value::Function(_)     → discriminant 6 = ValueType::Function (6)
Value::Userdata(_)     → discriminant 7 = ValueType::Userdata (7)
Value::Thread(_)       → discriminant 8 = ValueType::Thread (8)
```

**Add a compile-time test** using `std::mem::discriminant` or `as u8` casts to verify this mapping.

## Required Tests

Create `crates/lua_core/tests/value_tests.rs` with these test cases:

```rust
use lua_core::types::ValueType;
use lua_core::value::Value;

// ── Discriminant order ──────────────────────────────────────────
#[test]
fn test_value_discriminant_order() {
    // Verify that Value enum discriminant order matches ValueType
    assert_eq!(std::mem::discriminant(&Value::Nil) as u8, ValueType::Nil as u8);
    assert_eq!(std::mem::discriminant(&Value::Boolean(true)) as u8, ValueType::Boolean as u8);
    assert_eq!(std::mem::discriminant(&Value::Number(0.0)) as u8, ValueType::Number as u8);
    assert_eq!(std::mem::discriminant(&Value::String(std::ptr::null())) as u8, ValueType::String as u8);
    // ... etc for all 9 types
}

// ── Type checking ───────────────────────────────────────────────
#[test]
fn test_value_nil_default() { assert!(Value::Nil.is_nil()); }

#[test]
fn test_value_boolean_roundtrip() {
    let v = Value::Boolean(true);
    assert!(v.is_boolean());
    assert_eq!(v.as_boolean(), true);
}

#[test]
fn test_value_number_roundtrip() {
    let v = Value::Number(3.14);
    assert!(v.is_number());
    assert!((v.as_number() - 3.14).abs() < f64::EPSILON);
}

// ── Lua truthiness ──────────────────────────────────────────────
#[test]
fn test_lua_truthiness_nil_is_falsy() { assert!(Value::Nil.is_false()); }

#[test]
fn test_lua_truthiness_false_is_falsy() { assert!(Value::Boolean(false).is_false()); }

#[test]
fn test_lua_truthiness_true_is_truthy() { assert!(Value::Boolean(true).is_true()); }

#[test]
fn test_lua_truthiness_zero_is_truthy() { assert!(Value::Number(0.0).is_true()); }

#[test]
fn test_lua_truthiness_empty_string_is_truthy() {
    // Even null GC pointer is truthy (placeholder behavior until Phase 1.2)
    assert!(Value::String(std::ptr::null()).is_true());
}

// ── Equality ────────────────────────────────────────────────────
#[test]
fn test_value_equality_same_type_same_value() {
    assert_eq!(Value::Number(1.0), Value::Number(1.0));
    assert_eq!(Value::Boolean(false), Value::Boolean(false));
}

#[test]
fn test_value_equality_different_type_not_equal() {
    assert_ne!(Value::Nil, Value::Boolean(false));
}

#[test]
fn test_value_equality_gc_pointer_identity() {
    // Same pointer → equal
    let p = 0x42 as *const lua_core::types::GcString;
    assert_eq!(Value::String(p), Value::String(p));
    // Different pointers → not equal
    assert_ne!(Value::String(p), Value::String(std::ptr::null()));
}

// ── Display / toString ─────────────────────────────────────────
#[test]
fn test_display_nil() { assert_eq!(format!("{}", Value::Nil), "nil"); }

#[test]
fn test_display_boolean() {
    assert_eq!(format!("{}", Value::Boolean(true)), "true");
    assert_eq!(format!("{}", Value::Boolean(false)), "false");
}

#[test]
fn test_display_number() {
    assert_eq!(format!("{}", Value::Number(3.14)), "3.140000");
}

// ── Safe accessors ──────────────────────────────────────────────
#[test]
fn test_try_as_boolean_on_boolean() {
    assert_eq!(Value::Boolean(true).try_as_boolean(), Some(true));
}

#[test]
fn test_try_as_boolean_on_nil() {
    assert_eq!(Value::Nil.try_as_boolean(), None);
}

// ── Size constraint ─────────────────────────────────────────────
#[test]
fn test_value_size() {
    // Value must fit in 16 bytes on 64-bit (same as C++ std::variant)
    assert!(std::mem::size_of::<Value>() <= 16);
}
```

## Architecture Notes

- **`Value` should NOT implement `Copy`** — future `GcRef<T>` will need ownership tracking
- **`Value` SHOULD implement `Clone`** — placeholder `*const T` is Copy, but future proofing
- **`PartialEq` compares pointer equality for GC types** — matches C++ behavior
- **No `unsafe` blocks expected in this task** — only placeholder raw pointers, no deref
